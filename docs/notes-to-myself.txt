### gd
- The built-in `gd` command stands for "go to local declaration". It is a simple text-based search, which looks backwards from the cursor and finds the first occurance.
- It has no understanding of Python syntax, scope, or imports.
- It is not truly "go to definition" - just "go to first match"


### lsp
- Language Server Protocol, is a communication protocol between an editor, i.e. client, (like neovim) and a language-aware program, like pyright.
- The server runs as a background process because: it needs to parse and index the entire project, maintain in-memory model of the codebase, watches for file changes and updates its understanding, this is too heavy/slow to do on every keypress.
- Neovim has a built-in lsp client, so we just need to install a language server (an external program) and tell Neovim to start it and connect to it.
- List of LSP servers:
  - There's no "official" list since LSP is a protocol anyone can implement, but the main resources are these!
  - [Microsoft's list](https://microsoft.github.io/language-server-protocol/implementors/servers/)
  - [nvim-lspconfig's list](https://github.com/neovim/nvim-lspconfig/blob/master/doc/configs.md)


### importance of "file watching"
- After the following commit, `03afa45`, I noticed that there was a drastic degradation in speed in `MacMini M4` devices.
- Symptoms:
  - in large python projects, open a file & instantly do `jkhl` -> neovim becomes unresponsive for ~30s
  - same when trying to quit, `:q`, takes ~30s
- Cause:
  - pyright was registering a file watcher for ALL files in my workspace (**), plus watching the entire Python standard library. On m4 this was apparently causing the massive freeze. -> { globPattern = "**", kind = 7 }
- Solution:
  - disabled file watching capability for pyright in `lua/plugins/lsp.lua`
  - another optional fix was to add `pyrightconfig.json` to the workingspace, which potentially tells Pyright explicitly what to include/exclude, preventing it from scanning unnecessary files.revents pyright to watch
  ```json
  {
    "include": [
      "**/*.py"
    ],
    "exclude": [
      "**/node_modules",
      "**/__pycache__",
      "**/.*",
      "env",
      "venv",
      ".venv"
    ],
    "reportMissingImports": true,
    "reportMissingTypeStubs": false,
    "pythonVersion": "3.11"
  }
  ```

### LSP diagnostic display
- vim, by default has access to diagnostic and we can configure how to manage it through `vim.diagnostic.config()`. To display rich message of a warning/error in neovim, I mapped a key to trigger vim.diagnostic.open_float() and added to lsp.lua during LSPAttach.
